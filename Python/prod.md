
## Gunicorn

**Gunicorn** — это HTTP-сервер с поддержкой стандарта WSGI (Web Server Gateway Interface), предназначенный для работы с Python-приложениями. Он обрабатывает входящие HTTP-запросы, запускает приложение в отдельных процессах и распределяет их между рабочими потоками (воркерами). Gunicorn совместим с различными веб-фреймворками, прост в настройке, требует мало ресурсов сервера и относительно быстр. 

**Некоторые возможности Gunicorn:**

- поддержка многопоточности и асинхронных рабочих процессов (с использованием gevent или eventlet);
- возможность настройки количества воркеров в зависимости от требований приложения;
- поддержка hot reload воркеров без остановки сервиса.

## пособы запуска Gunicorn:

#### Шаг 1. Создаем файл сервиса

```bash
sudo nano /etc/systemd/system/gunicorn.service
```

#### Шаг 2. Копируем и вставляем этот текст

```js
[Unit]
Description=gunicorn daemon for cncwisdom
After=network.target
[Service]
User=django_user
Group=www-data
WorkingDirectory=/var/www/cncwisdom
ExecStart=/var/www/cncwisdom/venv/bin/gunicorn --workers 3 --bind 0.0.0.0:8000 cnc_wisdom.wsgi:application
Restart=on-failure
[Install]
WantedBy=multi-user.target
```
#### Шаг 3. Запускаем сервис
```bash
sudo systemctl daemon-reload
sudo systemctl start gunicorn
sudo systemctl enable gunicorn
sudo systemctl status gunicorn
```

#### Остановка сервиса
```bash
sudo systemctl stop gunicorn
```

## Nginx

**Nginx** — это веб-сервер с открытым исходным кодом, который может выполнять несколько функций:

- **Обратный прокси-сервер (reverse proxy).** Nginx принимает запросы от клиентов, перенаправляет их на сервер приложений (например, Gunicorn), получает ответ и возвращает его клиенту.
- **Обслуживание статических файлов.** Nginx может самостоятельно обслуживать статический контент (изображения, CSS, JavaScript), снижая нагрузку на сервер приложений.
- **Балансировка нагрузки.** Nginx может распределять запросы между несколькими экземплярами Gunicorn или другими серверами. 
- **SSL/TLS-терминация.** Nginx поддерживает шифрование и дешифрование данных, что позволяет использовать HTTPS без нагрузки на бэкенд-серверы. 
- **Кэширование, сжатие данных (gzip/brotli), ограничение скорости запросов** и другие функции для повышения производительности и безопасности. 

## Взаимодействие Gunicorn и Nginx

**Общий принцип работы:**

1. Клиент отправляет запрос на Nginx.
2. Nginx обрабатывает запрос и перенаправляет его на Gunicorn.
3. Gunicorn выполняет код приложения, генерирует ответ и передаёт его обратно Nginx.
4. Nginx возвращает ответ клиенту.

## Основные проблемы безопасности :

1. ❌ Нет HTTPS (все данные передаются в открытом виде)
    
2. ❌ DEBUG = True (видны ошибки и детали сервера)
    
3. ❌ Нет ограничения на попытки входа в админку
    
4. ❌ Нет настроек заголовков безопасности
    
5. ❌ Нет резервного копирования
